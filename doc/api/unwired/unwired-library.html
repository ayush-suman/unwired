AuthManager<T>
AuthManager is the base class to store the authentication state and credentials of the application.
Cancellable<T>
This contains the Response of a request and a cancel method On making a request, it returns the Cancellable object immediately without waiting for the request to complete. The user can cancel the request by calling the cancel method before the response arrives. It is safe to call the cancel method, even after the response arrives. In that case, the cancel method will do nothing.
DebugHttpWorker
DebugHttpWorker is an implementation of HttpWorker that processes requests and prints the debug logs to the console. By default, this is only used in debug mode.
HttpWorker
HttpWorker is the base class for all the workers that process the requests. This can be extended to create your own implementation for processing requests.
NativeHttpWorker
This implementation of HttpWorker is used to handle http requests in a separate Isolate. By default, it is used by native devices as the web platform does not support Isolates.
Parser<T>
Extend this class to create your own Parser.
QueueManager<T>
A class that manages a queue of objects.
RequestCompleterStoreManager
A StoreManager to store request Completers with the request id as the key and Completer of the request as the value.
RequestHandler
RequestIdQueueManager
A QueueManager to store request ids. Used by RequestHandler to keep a track of ongoing requests.
RequestInfoStoreManager
A StoreManager to store request debug info with the request id as the key and a Map of request information as the value.
Response<D>
This class contains the response details of a request This includes the status of the response The data or the parsed body of the response The error if the request failed or the parsing failed and hasError which is true if the error is not null and isCancelled which is true if the request was cancelled by the user. If the request gets cancelled by the user, the data and error will be null, and the status will be -1.
StoreManager<K, V>
Stores objects of type V with keys of type K This is useful for storing values that can be fetched their known meta data such as an id, name etc.
TokenAuthManager
TokenAuthManager is an implementation of AuthManager which stores a String type auth object. This is can be used to store a JWT token, for example.
WebHttpWorker
An HttpWorker implementation for web platform. This is perhaps the simplest implementation of HttpWorker. You can override the default setting in RequestHandler to use this for both web and native platforms for its simplicity.
